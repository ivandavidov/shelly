# Financial Data — Ръководство за AI агенти

## Цел на директорията

Тази директория съдържа Python скриптове, които кодират финансовите данни от тримесечните PDF отчети и генерират CSV файлове за дашборда. Инфраструктурата е **company-agnostic** — поддържа произволен брой компании.

---

## Структура

```
financial_data/
├── shelly/                         ← данни за Шелли Груп АД
│   ├── extract_q1_2021.py
│   ├── extract_q2_2021.py
│   ├── extract_q3_2021.py
│   ├── extract_q4_2021.py
│   ├── ...                         (по 4 файла за всяка година)
│   ├── extract_q4_2025.py
│   └── output/                     ← генерирани CSV файлове
│       ├── shelly_annual.csv
│       ├── shelly_quarterly_2021.csv
│       ├── ...
│       └── shelly_quarterly_all.csv
│
├── <company_id>/                   ← нова компания (по същия образец)
│   ├── extract_q1_YYYY.py
│   └── output/
│
├── generate_csv.py                 ← общ генератор (приема --company <id>)
├── validate.py                     ← общ валидатор (приема --company <id>)
└── AGENTS.md
```

Изходният файл `<company>_quarterly_all.csv` се **копира автоматично** в
`docs/companies/<company>/data.csv` след всяко генериране.

---

## Използване

```bash
# Генериране на CSV за Шелли
python3 generate_csv.py --company shelly
python3 generate_csv.py shelly        # positional аргумент — еквивалентно

# Валидация на данните за Шелли
python3 validate.py --company shelly
python3 validate.py shelly

# Нова компания (след създаване на extract файловете)
python3 generate_csv.py --company acme
python3 validate.py --company acme
```

## Как работят скриптовете

### extract_qX_YYYY.py

Всеки скрипт кодира данните от **един конкретен PDF отчет**. Данните са въведени ръчно след прочит на PDF-а (не се парсва автоматично).

Структура за Q1, Q2, Q3:
```python
income_statement = { "приходи_от_продажби": 51_759, ... }
balance_sheet    = { "общо_активи": 185_623, ... }
cash_flows       = { "нетни_потоци_оперативна": 8_072, ... }
meta             = { "период": "Q1 2025 (3М)", "края_на_период": "2025-03-31", ... }

def get_data():
    return {"meta": meta, "income_statement": ..., "balance_sheet": ..., "cash_flows": ...}
```

**Q4 скриптът е различен** — съдържа и сравнителните данни от предходната година:
```python
def get_data():
    return {
        "meta": ...,
        "2025": {"income_statement": ..., "balance_sheet": ..., "cash_flows": ...},
        "2024": {"income_statement": ..., "balance_sheet": ..., "cash_flows": ...},
    }
```

### generate_csv.py

- Открива автоматично наличните години от имената на `extract_q*_*.py` файловете — **не е нужен ръчен списък с години**.
- Зарежда динамично скриптовете чрез `importlib`.
- Скриптове, които не съществуват, се пропускат — колоните им остават празни.
- Генерира трите CSV файла и ги копира в `docs/companies/<company>/data.csv`.

### validate.py

- Прилага 13 счетоводни проверки (BS1–BS3, IS1–IS6, CF1–CF3) върху всеки наличен период.
- Допустима разлика: ±1 хил. лв. (за закръгляване в отчетите).

## Добавяне на нова компания

1. Създайте директория: `financial_data/<company_id>/`
2. Създайте `extract_q1_YYYY.py` … `extract_q4_YYYY.py` по образеца на Шелли
3. Стартирайте: `python3 generate_csv.py --company <company_id>`
4. Създайте `docs/companies/<company_id>/meta.json` с метаданни за компанията
5. Дашбордът се зарежда на: `index.html?company=<company_id>`

---

## Добавяне на нова година за съществуваща компания

1. Прочетете PDF отчетите (само стр. 1–8 за всеки отчет).
2. Създайте `<company>/extract_q1_YYYY.py` … `<company>/extract_q4_YYYY.py` по образеца на съществуващите скриптове.
3. Стартирайте `python3 generate_csv.py --company <company>` — CSV файловете се обновяват автоматично.

**Важно за Q4 скрипта:** той трябва да съдържа данни за две години — текущата FY и сравнителната предходна година — под отделни ключове (`"YYYY"` и `"YYYY-1"`).

## Речник на ключовете

Всички данни използват български ключове (snake_case). Таблицата по-долу показва съответствието с английските термини:

| Ключ | Английски термин |
|------|-----------------|
| `приходи_от_продажби` | Revenue |
| `себестойност_на_продажбите` | Cost of goods sold (COGS) |
| `брутна_печалба` | Gross profit |
| `други_приходи_от_дейността` | Other operating income |
| `разходи_за_продажби` | Selling expenses |
| `административни_разходи` | Administrative expenses |
| `други_разходи_за_дейността` | Other operating expenses |
| `печалба_от_оперативна_дейност_ebit` | EBIT |
| `финансови_приходи` | Financial income |
| `финансови_разходи` | Financial expenses |
| `дял_асоциирани_дружества` | Share of profit of associates |
| `печалба_преди_данъци_ebt` | EBT |
| `разход_за_данъци` | Income tax expense |
| `нетна_печалба` | Net profit |
| `общо_oci` | Total other comprehensive income |
| `общо_всеобхватен_доход` | Total comprehensive income |
| `нетен_доход_на_акция_eps` | EPS (BGN) |
| `общо_активи` | Total assets |
| `пари_и_парични_еквиваленти` | Cash and cash equivalents |
| `общо_собствен_капитал` | Total equity |
| `нетни_потоци_оперативна` | Operating cash flow |
| `нетни_потоци_инвестиционна` | Investing cash flow |
| `нетни_потоци_финансова` | Financing cash flow |
| `данъци_възстановени_платени` | Taxes recovered/(paid), net |

## Единици и валута

- Всички суми са в **хиляди лева (хил. лв. / BGN thousands)** — освен ако в meta.json на компанията е указано друго.
- Разходите са записани като **отрицателни числа** (напр. COGS = -24 456).
- EPS е в лева на акция (не в хиляди).
- Конвертирането BGN → EUR се управлява от `docs/companies/<company>/meta.json`.

---

## Четене на PDF отчети

**Четете само първите 7–8 страници** от всеки отчет — там са всички финансови таблици (ОПД, баланс, парични потоци). Останалите страници съдържат пояснителни бележки и не са нужни за кодиране на данните.

---

## Особености и известни проблеми

### Q1/Q2 данъчен ред в паричните потоци
Q1 и Q2 отчетите показват данъците като единен нетен ред `данъци_възстановени_платени`. Q3 и Q4 отчетите го разбиват на два реда. В CSV се използва само нетният ред за сравнимост.

### Баланс при Q4 скриптове
`get_dicts()` разпознава автоматично дали данните са пакетирани по година (Q4 формат) или директно (Q1–Q3 формат).

### Автоматично засичане на години
`generate_csv.py` и `validate.py` сканират `extract_q*_*.py` файловете и засичат сами годините — при добавяне на нов скрипт не е нужно да се редактира нищо друго.

## Изходни файлове

| Файл | Описание | Брой колони |
|------|----------|-------------|
| `<company>_annual.csv` | FY стойности по година | 2 + N години |
| `<company>_quarterly_YYYY.csv` | YTD по тримесечие за 1 година | 2 + 4 |
| `<company>_quarterly_all.csv` | YTD по тримесечие за всички години | 2 + 4×N |

Всички CSV файлове са кодирани в **UTF-8 с BOM** (`utf-8-sig`) за коректно отваряне в Excel.
