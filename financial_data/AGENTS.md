# Financial Data — Ръководство за AI агенти

## Цел на директорията

Тази директория съдържа Python скриптове, които кодират финансовите данни на **Шелли Груп ЕД** от тримесечните PDF отчети и генерират CSV файлове за анализ.

---

## Структура

```
financial_data/
├── extract_q1_YYYY.py      ← данни от Q1 отчет (3М YTD)
├── extract_q2_YYYY.py      ← данни от Q2 отчет (6М YTD)
├── extract_q3_YYYY.py      ← данни от Q3 отчет (9М YTD)
├── extract_q4_YYYY.py      ← данни от Q4/годишен отчет (12М + сравнителни)
├── generate_csv.py         ← обединява всичко и записва CSV файловете
└── output/
    ├── shelly_annual.csv           ← годишни (FY) данни, колони: 2021…2025
    ├── shelly_quarterly_YYYY.csv   ← YTD данни само за година YYYY
    └── shelly_quarterly_all.csv    ← YTD данни за всички години (всички тримесечия)
```

---

## Как работят скриптовете

### extract_qX_YYYY.py

Всеки скрипт кодира данните от **един конкретен PDF отчет**. Данните са въведени ръчно след прочит на PDF-а (не се парсва автоматично).

Структура за Q1, Q2, Q3:
```python
income_statement = { "приходи_от_продажби": 51_759, ... }
balance_sheet    = { "общо_активи": 185_623, ... }
cash_flows       = { "нетни_потоци_оперативна": 8_072, ... }
meta             = { "период": "Q1 2025 (3М)", "края_на_период": "2025-03-31", ... }

def get_data():
    return {"meta": meta, "income_statement": ..., "balance_sheet": ..., "cash_flows": ...}
```

**Q4 скриптът е различен** — съдържа и сравнителните данни от предходната година:
```python
def get_data():
    return {
        "meta": ...,
        "2025": {"income_statement": ..., "balance_sheet": ..., "cash_flows": ...},
        "2024": {"income_statement": ..., "balance_sheet": ..., "cash_flows": ...},
    }
```

### generate_csv.py

- Зарежда динамично всички налични `extract_qX_YYYY.py` чрез `importlib`.
- Скриптове, които не съществуват, се пропускат — колоните им остават празни.
- Генерира трите CSV файла при всяко стартиране.

## Речник на ключовете

Всички данни използват български ключове (snake_case). Таблицата по-долу показва съответствието с английските термини:

| Ключ | Английски термин |
|------|-----------------|
| `приходи_от_продажби` | Revenue |
| `себестойност_на_продажбите` | Cost of goods sold (COGS) |
| `брутна_печалба` | Gross profit |
| `други_приходи_от_дейността` | Other operating income |
| `разходи_за_продажби` | Selling expenses |
| `административни_разходи` | Administrative expenses |
| `други_разходи_за_дейността` | Other operating expenses |
| `печалба_от_оперативна_дейност_ebit` | EBIT |
| `финансови_приходи` | Financial income |
| `финансови_разходи` | Financial expenses |
| `дял_асоциирани_дружества` | Share of profit of associates |
| `печалба_преди_данъци_ebt` | EBT |
| `разход_за_данъци` | Income tax expense |
| `нетна_печалба` | Net profit |
| `общо_oci` | Total other comprehensive income |
| `общо_всеобхватен_доход` | Total comprehensive income |
| `нетен_доход_на_акция_eps` | EPS (BGN) |
| `общо_активи` | Total assets |
| `пари_и_парични_еквиваленти` | Cash and cash equivalents |
| `общо_собствен_капитал` | Total equity |
| `нетни_потоци_оперативна` | Operating cash flow |
| `нетни_потоци_инвестиционна` | Investing cash flow |
| `нетни_потоци_финансова` | Financing cash flow |
| `данъци_възстановени_платени` | Taxes recovered/(paid), net |

---

## Единици и валута

- Всички суми са в **хиляди лева (хил. лв. / BGN thousands)**.
- Разходите са записани като **отрицателни числа** (напр. COGS = -24 456).
- EPS е в лева на акция (не в хиляди).

---

## Четене на PDF отчети

**Четете само първите 7–8 страници** от всеки отчет — там са всички финансови таблици (ОПД, баланс, парични потоци). Останалите страници съдържат пояснителни бележки и не са нужни за кодиране на данните.

---

## Добавяне на нова година

1. Прочетете PDF отчетите (само стр. 1–8 за всеки отчет).
2. Създайте `extract_q1_YYYY.py`, `extract_q2_YYYY.py`, `extract_q3_YYYY.py`, `extract_q4_YYYY.py` по образеца на съществуващите скриптове за 2025.
3. Добавете новата година в списъка `ALL_YEARS` в `generate_csv.py` ако е извън обхвата 2021–2025.
4. Стартирайте `python3 generate_csv.py` — трите CSV файла се обновяват автоматично.

**Важно за Q4 скрипта:** той трябва да съдържа данни за две години — текущата FY и сравнителната предходна година — под отделни ключове (`"YYYY"` и `"YYYY-1"`). Вижте `extract_q4_2025.py` за пример.

## Особености и известни проблеми

### Q1/Q2 данъчен ред в паричните потоци
Q1 и Q2 отчетите показват данъците като единен нетен ред `данъци_възстановени_платени`. Q3 и Q4 отчетите го разбиват на два реда (възстановени данъци + платен корпоративен данък). В CSV се използва само нетният ред за сравнимост.

### Липсващи данни за стари години
Колоните за 2021–2024 в `shelly_quarterly_all.csv` са празни докато не се създадат съответните скриптове.

### Баланс при Q4 скриптове
`get_dicts()` в `generate_csv.py` разпознава автоматично дали данните са пакетирани по година (Q4 формат) или директно (Q1–Q3 формат).

## Изходни файлове

| Файл | Описание | Брой колони |
|------|----------|-------------|
| `shelly_annual.csv` | FY стойности по година | 2 + 5 години = 7 |
| `shelly_quarterly_YYYY.csv` | YTD по тримесечие за 1 година | 2 + 4 = 6 |
| `shelly_quarterly_all.csv` | YTD по тримесечие за всички години | 2 + 20 = 22 |

Всички CSV файлове са кодирани в **UTF-8 с BOM** (`utf-8-sig`) за коректно отваряне в Excel.
